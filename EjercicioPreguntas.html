Introducción a Git y GitHub

Git

¿Qué es Git? Git es un sistema de control de versiones de código abierto y gratuito que permite a los desarrolladores realizar un seguimiento de los cambios en su código y colaborar en proyectos de manera efectiva.

Instalación de Git: Para instalar Git, puedes descargar el instalador desde la página oficial de Git y seguir las instrucciones de instalación.

Configuración inicial de Git: Una vez instalado Git, debes configurar tu nombre de usuario y dirección de correo electrónico con los siguientes comandos:

git config --global user.name "Tu Nombre"
git config --global user.email "tu.correo@example.com"

GitHub

¿Qué es GitHub? GitHub es una plataforma de desarrollo de software que ofrece alojamiento de repositorios Git, colaboración en proyectos y herramientas de desarrollo integradas.

Crear una cuenta en GitHub: Para crear una cuenta en GitHub, ve a la página de inicio de GitHub y sigue las instrucciones de registro.

Diferencias y similitudes entre Git y GitHub: Git es un sistema de control de versiones, mientras que GitHub es una plataforma que ofrece alojamiento de repositorios Git y herramientas de colaboración. Git se puede utilizar de manera local, mientras que GitHub es una plataforma en la nube que permite la colaboración en proyectos.

Creación de un Repositorio Local

Para crear un repositorio local, debes utilizar los siguientes comandos:

git init

Este comando inicializa un nuevo repositorio Git en la carpeta actual.

git status

Este comando muestra el estado actual del repositorio, incluyendo los archivos modificados y los archivos que no están siendo rastreados.

git add <file>

Este comando agrega un archivo específico al staging area.

git commit -m "Mensaje de commit"

Este comando crea un nuevo commit con los cambios en el staging area y agrega un mensaje de commit.

Estructura de un proyecto en Git: Un proyecto en Git se organiza en una carpeta raíz que contiene los archivos y carpetas del proyecto. La carpeta `.git` contiene los archivos de configuración y los datos del repositorio.
